# Contributing to BAIT

BAIT gets better every time someone asks their agent a new question. The main way to
contribute is to **save good answers as cookbook recipes** so the next person's agent can
reuse them.

## ⛔ The one hard rule

**Never commit data.** No raw rows, no real station coordinates, no `.rds`/`.csv`/`.xlsx`/
`.duckdb` files, no screenshots of data tables. Recipes contain *code and prose*, plus
rounded or synthetic example values where needed. The `.gitignore` blocks data extensions
as a safety net — do not loosen it.

## Adding a cookbook recipe

1. Copy [`cookbook/_TEMPLATE.md`](cookbook/_TEMPLATE.md) to `cookbook/<short-slug>.md`.
2. Fill in: the natural-language question(s) it answers, the R code, and the *shape* of the
   expected output (a description or a tiny rounded/synthetic example — not real data).
3. If the recipe is runnable end-to-end, add the matching script to
   [`examples/`](examples/) with the same slug.
4. Make sure the code follows house style: tidyverse, read-only DuckDB connection, lazy
   `tbl()` + `collect()` at the end, real column names from
   [`knowledge/field-glossary.md`](knowledge/field-glossary.md).
5. Open a pull request (or commit, if you have write access). Keep one recipe per PR.

### Letting your agent draft it

After your agent solves something new, just say *"save this as a BAIT recipe."* It will
copy the template, fill it in, and show you the draft. **You review it** — especially that
no real data leaked in — before committing.

## Improving skills or knowledge

- `skills/*/SKILL.md` — keep procedures short; link to `knowledge/` rather than duplicating.
- `knowledge/field-glossary.md` — **do not edit by hand.** It is generated by
  [`scripts/distill_xsd.R`](scripts/distill_xsd.R) from the NMD Biotic v3 XSD. Edit the
  script (or its hand-curated overrides) and regenerate.
- Keep [`CLAUDE.md`](CLAUDE.md) and [`AGENTS.md`](AGENTS.md) in sync — they are intentionally
  near-identical so every agent gets the same guardrails.

## Style

- Markdown wrapped at ~95 columns.
- R: tidyverse, meaningful names, comments matching the surrounding density.
- One logical change per PR; describe *why*, not just *what*.
